# World Cup Predictor 2026 - Development v4.1

גרסת development של ליגת ניחושים בעברית RTL, מותאמת Chrome ומובייל, עם שרת שיתופי אמיתי יותר, הרשאות לפי תפקיד, ניקוד מאוזן לשלבים המאוחרים וחיבור לייב מוכן לחיבור לספק תוצאות.

## מה חדש ב-v4
- התחברות פרטית עם שמירת סיסמאות כ-hash ו-session בצד השרת
- הפרדה בין מסך שחקן למסך אדמין
- שחקן רואה רק את הניחושים שלו ואת שמות התצוגה בליגה
- מסך הכניסה לא מציג רשימת משתמשים או סיסמאות
- 12 בתים ו-104 משחקי מונדיאל 2026 במבנה מלא
- משחקי ניסוי מעודכנים להיום: ארסנל-ספורטינג CP, באיירן-ריאל מדריד
- ניקוד חדש לשלבי ההכרעה:
  - כיוון נכון: 1
  - תוצאה מדויקת רגיל / בתים / שמינית / 32 האחרונות: 3
  - רבע גמר: 4
  - חצי גמר: 5
  - גמר: 6
- בונוס בתים מאוזן: רק מקום 1 ו-2 בכל בית, נקודה לכל מיקום נכון
- בונוס זוכה: 20
- בונוס מלך שערים: 15, או 20 אם נבחר `אחר` והזוכה לא ברשימה
- מאגר רחב של מועמדים למלכות שערים + `אחר`
- מסך אדמין לניהול משתמשים, איפוס סיסמאות, סנכרון לייב ועדכוני override
- תזכורות דפדפן שעה לפני משחק
- שכבת לייב מוכנה ל-API-Football או JSON generic

## מבנה הפרויקט
- `public/` - כל קבצי ה-frontend
- `server/server.mjs` - שרת Node יחיד שמגיש גם את ה-static וגם את ה-API
- `server/data/users.json` - משתמשים (hash בלבד)
- `server/data/state.json` - ניחושים, תוצאות ומצב משותף
- `.env.example` - משתני סביבה לדיפלוימנט ול-live provider

## הרצה מקומית
```bash
npm start
```

או:
```bash
cd server
npm start
```

ברירת המחדל: `http://localhost:8080`

## תיקון המסך הלבן ב-v4
בוצעו שלושה תיקונים כדי למנוע מצב של index לבן/ריק:
- מעבר לנתיבי assets יחסיים במקום absolute paths
- גישה בטוחה ל-`localStorage` גם בסביבות preview/sandbox
- fallback למסך שגיאה ברור במקום blank screen

אם פותחים את `public/index.html` ישירות, ה-frontend אמור לעלות. כדי להתחבר באמת צריך backend פעיל באותה כתובת.

## אדמין והתחברות ראשונה
ב-v4.1 אין יותר שם משתמש וסיסמה קשיחים בתוך ה-repo הציבורי.

מומלץ להגדיר בדיפלוימנט:
- `BOOTSTRAP_ADMIN_USERNAME=admin`
- `BOOTSTRAP_ADMIN_PASSWORD=סיסמה_חזקה_שתבחר`
- `BOOTSTRAP_ADMIN_DISPLAY_NAME=מנהל`

אם לא תגדיר `BOOTSTRAP_ADMIN_PASSWORD`, המערכת תייצר סיסמה חד-פעמית אקראית ותדפיס אותה ללוגים בזמן העלייה הראשונה.

ל-development מקומי מהיר אפשר גם להגדיר:
- `SEED_SAMPLE_USERS=true`

ואז יווצרו גם משתמשי דוגמה:
- `asaf / Wc2026!01`
- `player02` עד `player20` עם סיסמאות בתבנית `Wc2026!XX`

## איך לשתף עם חברים
1. לפרוס את השרת לכתובת ציבורית אחת.
2. להיכנס עם משתמש האדמין שהגדרת ב-`BOOTSTRAP_ADMIN_*`.
3. ליצור משתמש לכל חבר מתוך דאשבורד האדמין.
4. לשלוח לכל חבר רק את שם המשתמש והסיסמה שלו.
5. כל חבר נכנס לאותה כתובת, ומקבל מסך שחקן פרטי.

## חיבור לייב
כדי לחבר ספק תוצאות אמיתי:
1. להגדיר `LIVE_PROVIDER=api-football`
2. להוסיף `API_FOOTBALL_KEY`
3. אם רוצים גם Auto top scorer למונדיאל, להוסיף `WORLDCUP_LEAGUE_ID`

אפשר גם לחבר מקור JSON משלך:
1. `LIVE_PROVIDER=generic`
2. `GENERIC_LIVE_URL=https://...`

השרת יודע:
- למשוך לייב
- לנעול אוטומטית תוצאות סופיות
- לחשוף ניחושי כולם עם שריקת פתיחה
- לעדכן זוכה טורניר אוטומטית לפי הגמר

## הערות חשובות
- זו גרסת development לשיפור ועבודה משותפת, לא פרודקשן סופי.
- התזכורות כרגע הן התראות דפדפן. עבור Web Push ברקע מלא צריך להוסיף VAPID / ספק push.
- מאגר מלכי השערים הוא seed development editable.
- טבלאות הבתים משתמשות ב-tiebreaker פשוט: נקודות, הפרש שערים, שערי זכות, שם.
